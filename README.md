# Gemini CLI UI Capture Kit (Figma Sync & MCP Ready)

This repository provides a powerful, headless engine that captures running web interfaces and converts them into **High-Fidelity Vector Blueprints** natively inside Figma. 

Unlike traditional screenshot tools, this kit uses an Atomic Precision Scansion engine (via Puppeteer and the Range API) to capture actual text nodes, SVG icons, proper Z-indexing, and exact CSS properties, ensuring your layouts are perfectly editable in Figma.

It is designed to be the ultimate bridge between code generated by AI Agents (Claude, Cursor, Copilot, Gemini) and Design workflows.

---

## 🌟 Key Features

- **True Vectorization:** Converts DOM elements into native Figma Frames, Text Layers, and SVGs. No static pixelated images.
- **Atomic Precision (v3.1):** Uses the browser's Range API to place text exactly where it renders, preventing label/input overlapping.
- **Z-Order Awareness:** Respects complex stacking contexts (modals, absolute positioning, overlays).
- **UI Noise Filtering:** Automatically strips out decorative framework layers (e.g., `MuiTouchRipple`) that clutter design files.
- **Fully Dynamic Plugin:** The Figma plugin reads your captured data automatically without requiring manual code updates.
- **AI/MCP Ready:** Fully compatible with Model Context Protocol (MCP) workflows.

## 📦 Installation

Clone this repository outside of your main project to keep your workspaces clean:

```bash
git clone git@github.com:coviyarce/gemini-cli-ui-capture.git
cd gemini-cli-ui-capture
npm install
```

## ⚙️ Configuration

Before running the capture engine, configure the `capture-config.json` file in the root directory to point to your local development server and define the screens you want to capture.

```json
{
  "baseUrl": "http://localhost:3000",
  "screens": [
    { 
      "category": "Main Flow", 
      "id": "dashboard", 
      "name": "Dashboard Overview", 
      "selector": "body" 
    },
    { 
      "category": "Components", 
      "id": "settings-modal", 
      "name": "Settings Modal", 
      "actions": [
        { "type": "click", "text": "Open Settings" },
        { "type": "wait", "ms": 500 }
      ],
      "selector": ".modal-content" 
    }
  ]
}
```

*Pro Tip: For complex flows, build a "Storyboard" route in your app that renders all components side-by-side. This allows the tool to scan everything at once without complex interaction scripts.*

## 🚀 Usage

### 1. Run the Capture Engine
Ensure your local web app (e.g., React, Vue, Next.js) is running at the `baseUrl` specified in your config.

To build a fresh vector blueprint (Recommended):
```bash
node scripts/capture-to-figma.cjs --rebuild
```

### 2. Synchronize the Plugin
Once the capture is complete, inject the data into the local Figma plugin:
```bash
node scripts/update-plugin.cjs
```

### 3. Import into Figma
1. Open Figma desktop app.
2. Go to `Plugins > Development > Import plugin from manifest...`
3. Select the `figma-plugin/manifest.json` file from this downloaded repository.
4. Run the plugin. You will see a dynamically generated list of your captured screens.
5. Select resolutions (1080p/720p), pick your screens, and click **Import Selected**.

## 🤖 Integrating with AI Agents (Claude, Cursor, Copilot, Gemini)

This kit is heavily optimized to be used alongside modern AI coding assistants. Since these tools are excellent at writing code but cannot easily "see" the visual result, you can use this kit as a verification loop.

**The Workflow:**
1. **Prompt the AI:** Ask Cursor, Claude, or Gemini to build a UI component based on a description.
2. **Render Locally:** Let the AI save the file and run your local server.
3. **Capture:** Run this UI Capture Kit (`capture-to-figma.cjs`).
4. **Figma Validation:** Import the result into Figma. You can now visually inspect the AI's work natively, tweak the paddings or colors, and hand it back to the AI for further refinement.

### 🔌 Figma MCP Integration Guide

If you are using an AI that supports the **Model Context Protocol (MCP)** (like Claude Desktop, Cursor, or Gemini CLI), you can integrate Figma directly so the AI can read your designs and write the code for you before you use this capture kit to send it back.

**Step 1: Get your Figma Personal Access Token**
1. Log into Figma.
2. Go to your Account Settings (click your avatar top left -> Settings).
3. Scroll down to **Personal access tokens**.
4. Click **Generate new token**, give it a name (e.g., "MCP Token"), set expiration, and copy the string. *Keep this safe.*

**Step 2: Get your Figma File Key**
1. Open the Figma file containing your designs.
2. Look at the URL. It looks like: `https://www.figma.com/design/THIS_IS_THE_FILE_KEY/File-Name`
3. Copy the alphanumeric string after `/design/`.

**Step 3: Setup the MCP Server**
Configure your AI tool (e.g., in Claude Desktop's `claude_desktop_config.json` or Gemini CLI's tools config) to use the Figma MCP server.

```json
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "@figma/mcp"],
      "env": {
        "FIGMA_ACCESS_TOKEN": "YOUR_PERSONAL_ACCESS_TOKEN"
      }
    }
  }
}
```

Once connected, you can tell your AI:
> "Look at Figma node ID `123:456` in file `THIS_IS_THE_FILE_KEY`. Build this component in React."

After the AI builds it, you use this **UI Capture Kit** to send the rendered code back to Figma to verify accuracy!
